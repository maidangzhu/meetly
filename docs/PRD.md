# PRD: fuck-stupid-interview MVP

## 1. 产品定义

`fuck-stupid-interview` 是一个桌面端实时会议/面试辅助工具。它以一个不打断当前工作的悬浮灵动岛存在于屏幕顶部，在用户参加中文会议、面试、销售沟通、客户访谈时，实时监听系统音频并转写重点，在用户需要时给出一句可以直接说出口的回答建议。

第一版不是完整的会议知识库，也不是自动代替用户开会的机器人。第一版只解决一个核心问题：

> 用户正在高压对话里，一边听对方说话，一边需要快速组织更清晰、更稳妥的回答。

## 2. 目标用户

### 2.1 首要用户

- 正在远程面试的候选人。
- 需要频繁参加英文/中文会议但表达压力大的职场用户。
- 销售、客户成功、咨询顾问等高频沟通岗位。

### 2.2 使用环境

- macOS 桌面端优先。
- 用户通常同时打开 Zoom、腾讯会议、飞书会议、Google Meet、浏览器、文档或简历。
- 用户不希望会议窗口被遮挡，不希望切换应用，不希望让对方看到辅助工具。

## 3. 产品目标

### 3.1 P0 目标

1. 以一个原生悬浮灵动岛常驻在屏幕顶部。
2. 实时捕获系统音频并转写最近对话。
3. 用户点击或按快捷键时，基于最近 1-3 分钟转写内容生成可直接回应的建议。
4. 支持截图分析：在截图前自动隐藏灵动岛，结合屏幕内容与最近对话生成建议。
5. 第一版必须包含隐藏/录屏规避能力，但产品口径必须是“尽量不被常见录屏和截图采集到”，不承诺绝对不可见。
6. 配置国内实时 STT 与 LLM API Key，能在中国网络环境下跑通。

### 3.2 非目标

第一版不做：

- 多端同步。
- 团队空间。
- 历史会议知识库/RAG。
- 日历集成。
- 浏览器扩展。
- 自动填写、自动发送消息。
- 本地 Whisper 安装器。
- 完整说话人分离。
- 完整反作弊或绕过检测能力。
- 全平台一致隐藏承诺。

## 4. MVP 范围

### 4.1 必须有

- 原生悬浮灵动岛。
- 展开/收起。
- 开始/暂停监听。
- 实时转写流。
- 最近上下文缓存。
- 一键生成回答建议。
- 截图后提问。
- 设置页。
- 全局快捷键。
- 基础隐藏能力。
- 连接状态和错误状态。

### 4.2 可以简化

- 第一版只采系统音频，不采麦克风。
- 第一版只支持一个 STT Provider：阿里云百炼/Model Studio 实时语音识别。
- LLM Provider 可以先走 OpenAI-compatible API 格式，允许用户填 base URL、model、API Key。
- 转写不强求完整标点和说话人识别，只要能稳定给 AI 提供上下文。

### 4.3 预留但不实现

- 本地 Whisper。
- 腾讯云、火山、讯飞、百度 STT 适配。
- 本地小模型建议。
- 会议记录导出。
- Persona 模板市场。
- 多屏幕独立位置。

## 5. 核心体验

### 5.1 默认状态

灵动岛位于屏幕顶部中央，默认尺寸约 `600 x 54`。

交互和样式第一版明确对标 Pluely：

- 顶部居中的紧凑横向工具条。
- 透明、无边框、圆角、轻边框、轻阴影。
- 主体是一张 `Card` 式容器，不做 dashboard。
- 左侧是系统音频/监听入口。
- 中间是 Ask 输入、转写 ticker 或音频可视化。
- 右侧是截图、状态、设置/更新、拖拽手柄。
- 展开内容从灵动岛下方出现，不跳转主界面。

默认状态只显示：

- 监听状态点。
- 当前识别短句或“Listening”。
- Ask 按钮。
- 截图按钮。
- 展开按钮。

视觉原则：

- 它必须像系统级浮层，而不是一个网页卡片。
- 第一版必须贴近 Pluely 的密度、尺寸、圆角、按钮比例和 popover 行为。
- 不做大面积渐变。
- 不做营销式 hero。
- 不遮挡主会议窗口重点区域。
- 字号克制，信息密度高。
- 所有按钮必须可被快速理解，优先图标 + tooltip。

### 5.2 展开状态

用户点击展开或按快捷键后，窗口高度扩展到约 `600 x 420` 或 `600 x 600`。

展开区域包含：

- 最近转写列表。
- 当前 AI 建议。
- 快速模式切换。
- 截图预览缩略状态。
- 设置入口。

展开不是一个完整 dashboard，而是一个临场操作面板。

展开交互也对标 Pluely：

- 通过 popover/panel 从工具条下方展开。
- 工具条仍保留在顶部。
- 面板宽度跟随窗口宽度，第一版保持 `600px`。
- 面板内容使用紧凑 header + scroll area。
- 关闭 popover 后窗口自动回到 `600 x 54`，但正在监听或生成时不能误收起关键状态。

### 5.3 回答建议

用户触发 Ask 后，系统根据最近上下文生成建议。

建议输出必须短：

- 第一行：推荐回答，一句话或两句话。
- 第二块：可选补充要点，最多 3 条。
- 第三块：如果问题不清楚，给一个澄清问题。

不允许第一版默认输出长篇面试教程。临场用户没有时间读。

### 5.4 截图分析

用户触发截图时：

1. 灵动岛立即隐藏。
2. 延迟约 120-200ms。
3. Rust 侧截图当前屏幕。
4. 灵动岛恢复。
5. 把截图和最近转写发送给 LLM。
6. 返回面向当前屏幕的建议。

截图场景包括：

- 面试题页面。
- 共享屏幕里的问题。
- 代码题、case 题、销售材料。

### 5.5 设置

设置页可以作为独立窗口或灵动岛展开后的子视图。第一版建议做独立普通窗口，避免把所有配置塞进小浮层。

必须配置：

- STT Provider API Key。
- STT 模型。
- LLM base URL。
- LLM API Key。
- LLM model。
- 隐藏策略开关。
- 快捷键。
- 开机启动可先不做。

## 6. 信息架构

### 6.1 主灵动岛

- `Status`: idle/listening/thinking/error。
- `TranscriptTicker`: 最新识别片段。
- `AskAction`: 生成建议。
- `CaptureAction`: 截图分析。
- `ExpandAction`: 展开。

### 6.2 展开面板

- `TranscriptPanel`: 最近转写。
- `SuggestionPanel`: AI 建议。
- `ContextPanel`: 最近截图/当前任务。
- `QuickMode`: interview/meeting/sales/debug。
- `Controls`: pause/clear/settings。

### 6.3 设置窗口

- `Providers`: STT/LLM。
- `Audio`: 系统音频设备与权限。
- `Stealth`: 内容保护与截图前隐藏。
- `Shortcuts`: 快捷键。
- `Diagnostics`: 连接测试、权限测试、录屏保护测试。

## 7. 用户流程

### 7.1 首次启动

1. 用户打开应用。
2. 显示设置窗口，引导配置 STT 与 LLM。
3. 请求系统音频/屏幕录制相关权限。
4. 运行连接测试。
5. 进入灵动岛默认状态。

首启必须避免“空灵动岛不知道为什么不能用”的体验。

### 7.2 正常会议

1. 用户进入会议软件。
2. 点击灵动岛开始监听或使用快捷键。
3. 灵动岛显示实时转写短句。
4. 对方问问题。
5. 用户按快捷键 Ask。
6. 灵动岛展开并显示短建议。
7. 用户口头回答。

### 7.3 面试题/屏幕题

1. 用户看到题目。
2. 按截图快捷键。
3. 应用隐藏灵动岛并截图。
4. 结合截图和最近对话生成建议。
5. 用户看到“回答结构/下一步操作/注意点”。

### 7.4 出错恢复

如果 STT 断开：

- 灵动岛状态点变为黄色/红色。
- 展开后显示“语音识别已断开，点击重连”。
- 不弹系统级大窗口打断会议。

如果音频权限缺失：

- 灵动岛显示不可监听。
- 设置页给出明确权限检查项。

如果 LLM 超时：

- 展示“生成超时，保留当前转写，可重试”。

## 8. 快捷键

默认建议：

- `Cmd/Ctrl + Shift + Space`: 显示/隐藏灵动岛。
- `Cmd/Ctrl + Shift + L`: 开始/暂停监听。
- `Cmd/Ctrl + Shift + A`: Ask。
- `Cmd/Ctrl + Shift + S`: 截图分析。
- `Cmd/Ctrl + Shift + Up/Down`: 展开/收起。

所有快捷键必须可改，因为会议软件和 IDE 很容易冲突。

## 9. 隐藏与录屏保护的产品口径

第一版必须实现隐藏，但不做虚假承诺。

产品内推荐文案：

> 已启用隐藏模式：应用会尽量避免悬浮窗被常见截图、录屏和本应用截图链路捕获。不同系统版本和录屏软件实现不同，无法保证所有场景都不可见。

验收口径：

- 自己的截图链路必须 100% 不包含灵动岛。
- macOS 常见截图路径尽量不捕获窗口内容。
- Windows 10 2004+ 使用系统 API 排除窗口捕获。
- 第三方会议软件/录屏软件以测试矩阵结果为准，不承诺绝对。

## 10. 权限与隐私

第一版默认 local-first 存储：

- API Key 保存在本机安全存储。
- 转写上下文默认只保留本地最近缓存。
- 不做云端用户账号。
- 不上传完整会议记录到自有服务器。

会发送到第三方服务的数据：

- 音频片段发送到 STT Provider。
- 最近转写和截图发送到 LLM Provider。

设置页必须明确展示这一点。

## 11. 成功指标

MVP 不追求 DAU，先看核心链路是否成立。

### 11.1 技术指标

- 启动到灵动岛出现：小于 2 秒。
- Ask 首 token 延迟：小于 4 秒。
- STT 识别延迟：目标 1-2 秒内有增量结果。
- 截图分析完整返回：小于 8 秒。
- 连续监听 30 分钟不崩溃。

### 11.2 体验指标

- 用户不需要切换应用即可完成 Ask。
- 用户能在 3 秒内理解建议。
- 用户在真实会议中不会频繁误触。
- 灵动岛收起后不显著遮挡会议内容。

## 12. 验收标准

### 12.1 基础链路

- 启动后出现灵动岛。
- 灵动岛默认形态符合 Pluely-style：`600 x 54`、横向工具条、图标按钮、拖拽手柄、轻透明 Card。
- 可以开始/停止监听。
- 系统会议音频能被转写。
- 转写能滚动显示最近内容。
- Ask 能基于最近转写生成建议。
- 截图分析能返回建议。
- 设置项重启后保留。

### 12.2 隐藏链路

- 灵动岛设置 `contentProtected`。
- macOS 转为 NSPanel 或等价原生浮层。
- 自己截图前隐藏，截图后恢复。
- Windows 使用 `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`。
- 提供一键测试截图，用户能看到测试结果。

### 12.3 错误链路

- STT Key 错误时能提示。
- LLM Key 错误时能提示。
- 音频捕获失败时不崩溃。
- 网络断开后可重连。
